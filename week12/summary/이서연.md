# 리스트
- 데이터에 순서를 매겨 늘어놓은 자료구조
# 연결 리스트 
- 노드 : 각각의 원소, 데이터와 포인터 가지고 있음
- 포인터 : 뒤쪽 노드를 참조
- 머리노드(맨앞) - 꼬리노드(맨뒤)
- 앞쪽노드 - 뒤쪽노드

- 배열로 노드를 만들면 데이터를 삽입, 삭제함에 따라 옮겨야 하므로 비효율적
-> 노드용 인스턴스 생성, 삭제 시 노드용 인스턴스를 없애는 것으로 해결 

### 노드 클래스
- data + next(뒤쪽 포인터)

```
배열 : 영화관 좌석 지정
- 몇번째 자리인지 바로 앎, 
- 자리 하나 끼워넣을때 다 밀어야 함
연결 리스트 : 기차 칸 
- 각 캉니 다음 칸 위치를 알고 있음.
- 중간에 칸 하나 끼울 때 앞뒤 칸 연결만 바꾸면 됨.
- 5번째 칸으로 가려면 1번부터 순서대로 걸어가야 함
```
