# 해시법

검색과 더불어 데이터의 추가·삭제도 효율적으로 수행할 수 있는 알고리즘

- **해시(Hash)** : '긁어모음, 뒤죽박죽, 가늘게 썬 고기 음식'을 뜻함
- 충돌이 전혀 발생하지 않는다면 시간 복잡도는 **O(1)**
<br/><br/>

## 해시법이란?

'데이터를 저장할 위치 = 인덱스'를 간단한 연산으로 구하는 것 <br/>
원소의 검색뿐 아니라 추가·삭제도 효율적으로 수행할 수 있다.
<br/><br/>

## 해시 충돌

키와 해시값은 일반적으로 **n:1** 관계이다. <br/>
저장할 버킷이 중복되는 현상을 **충돌(Collision)** 이라고 한다.

#### 충돌을 해결하기 위한 두 가지 방법:

| 방법 | 설명 |
|------|------|
| **체인법** | 해시값이 같은 원소를 연결 리스트로 관리 |
| **오픈 주소법** | 빈 버킷을 찾을 때까지 해시를 반복 |

<br/><br/>
---

## 체인법

해시값이 같은 데이터를 체인 모양의 **연결 리스트**로 연결하는 방법.  
**오픈 해시법**이라고도 한다.

> 배열의 각 버킷에 저장하는 것은, 인덱스를 해시값으로 하는 연결 리스트의 앞쪽 노드를 참조하는 것이다.

<br/>

### Node 클래스

개별 버킷을 나타내는 클래스.

| 필드 | 자료형 | 설명 |
|------|--------|------|
| `key` | 임의의 자료형 | 키 |
| `value` | 임의의 자료형 | 값 |
| `next` | Node형 | 뒤쪽 노드를 참조 |

<br/>

### ChainedHash 클래스

해시 테이블 전체를 관리하는 클래스.

| 필드 | 자료형 | 설명 |
|------|--------|------|
| `capacity` | int | 해시 테이블의 크기 (배열 `table`의 원소 수) |
| `table` | list | 해시 테이블을 저장하는 배열 |

<br/>

### 해시값 계산 관련 함수

#### `sha256` 알고리즘
RSA의 FIPS 알고리즘을 바탕으로 함<br/> 
주어진 바이트 문자열의 해시값을 구하는 해시 알고리즘의 생성자.  
`hashlib` 모듈은 `sha256` 외에도 MD5 알고리즘인 `md5` 등 다양한 해시 알고리즘을 제공한다.

| 함수 | 설명 |
|------|------|
| `encode()` | `hashlib.sha256`에 전달하기 위해 `str`형 문자열을 바이트 문자열로 변환 |
| `hexdigest()` | sha256 알고리즘에서 해시값을 **16진수 문자열**로 반환 |
| `int()` | `hexdigest()`로 꺼낸 문자열을 **16진수 기반 int형**으로 변환 |

<br/><br/>

## 오픈 주소법

충돌이 발생했을 때 **재해시(Rehashing)** 를 수행하여 빈 버킷을 찾는 방법.  
**닫힌 해시법(Closed Hashing)** 이라고도 한다.
<br/><br/>

### Bucket 클래스

개별 버킷의 상태를 포함하는 클래스.

| 필드 | 자료형 | 설명 |
|------|--------|------|
| `key` | 임의의 자료형 | 키 |
| `value` | 임의의 자료형 | 값 |
| `stat` | Status형 | 버킷의 현재 상태 |

<br/>

### 버킷 상태 (Status 열거형)

| 상태 | 설명 |
|------|------|
| `OCCUPIED` | 데이터가 존재함 |
| `EMPTY` | 데이터가 없음 |
| `DELETED` | 데이터가 삭제됨 (탐색 시 건너뜀) |

<br/>

### 재해시(Rehashing)

충돌 발생 시 새로운 해시값을 계산하여 빈 버킷을 탐색하는 과정.  
선형 조사법의 경우 아래와 같이 다음 인덱스를 순차적으로 탐색한다.

```python
# 재해시: 현재 해시값에서 +1 이동
rehash_value = (hash_value + 1) % capacity
```

> `DELETED` 상태의 버킷은 탐색 시 건너뛰되, 새 데이터 추가 시에는 재사용 가능하다.
