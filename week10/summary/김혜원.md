
# 셸 정렬

- 단순 삽입 정렬의 장점을 살리고 단점을 보완한 알고리즘
- 배열 원소를 **일정 간격(gap)** 으로 그룹을 나눠 각 그룹별로 삽입 정렬을 수행하고, gap을 점차 줄여가며 이 과정을 반복한다.
- **시간 복잡도:** O(n¹·²⁵)


```python
def shell_sort(a):
    n = len(a)
    h = n // 2
    while h > 0:
        for i in range(h, n):
            j, tmp = i - h, a[i]
            while j >= 0 and a[j] > tmp:
                a[j + h] = a[j]
                j -= h
            a[j + h] = tmp
        h //= 2
```

## 개선된 셸 정렬

**문제점** : 기본 셸 정렬의 gap(8→4→2→1)은 배수 관계라 같은 그룹의 원소만 반복 비교될 수 있음 <br/> 
이를 해결하기 위해 `h = h×3 + 1` 수열을 사용해 원소가 더 고르게 이동하도록 개선한다.

```python
def shell_sort(a):
    n = len(a)
    h = 1
    while h < n // 9:
        h = h * 3 + 1
    while h > 0:
        for i in range(h, n):
            j, tmp = i - h, a[i]
            while j >= 0 and a[j] > tmp:
                a[j + h] = a[j]
                j -= h
            a[j + h] = tmp
        h //= 3
```

# 퀵 정렬

일반적으로 가장 빠른 정렬 알고리즘 <br/>
**피벗(pivot)** 을 기준으로 작은 값은 왼쪽, 큰 값은 오른쪽에 배치한 뒤 재귀적으로 정렬한다.

- **평균 시간 복잡도:** O(n log n)
- **최악 시간 복잡도:** O(n²) 
- 
```python
def qsort(a, left, right):
    pl, pr = left, right
    x = a[(left + right) // 2]  # 피벗: 중앙 원소
    while pl <= pr:
        while a[pl] < x: pl += 1
        while a[pr] > x: pr -= 1
        if pl <= pr:
            a[pl], a[pr] = a[pr], a[pl]
            pl += 1; pr -= 1
    if left < pr: qsort(a, left, pr)
    if pl < right: qsort(a, pl, right)
```

### 피벗 선택 전략
 
피벗 선택 방법은 퀵 정렬의 실행 효율에 큰 영향을 미친다.
 
맨 앞 원소를 피벗으로 쓰면 이미 정렬된 배열에서 분할이 한쪽으로 완전히 치우쳐 빠른 정렬을 기대할 수 없다. <br/>
가장 이상적인 피벗은 전체의 중앙값이지만, 이를 구하는 비용이 크다.
 
이 문제를 피하기 위한 두 가지 방법:
 
**방법 1** — 나누어야 할 배열의 원소 수가 3 이상이면, 배열에서 임의의 원소 3개를 꺼내 중앙값을 피벗으로 선택한다.<br/> 
예를 들어 `[8,7,6,5,4,3,2,1,0]`에서 맨 앞(8), 가운데(4), 맨 끝(0) 중 중앙값인 **4를 피벗**으로 선택하면 치우치지 않고 나눌 수 있다.
 
**방법 2 (방법 1의 개선)** — 맨 앞, 가운데, 맨 끝 3개를 정렬한 뒤 가운데 원소와 맨 끝에서 두 번째 원소를 선택한다.  <br/>
맨 끝에서 두 번째 값인 `a[right-1]`이 피벗으로 선택되고, 동시에 나눌 대상을 `a[left+1] ~ a[right-2]`로 좁힌다.

<br/>

> 파이썬 기본 제공: `sorted()` / `list.sort()`

# 병합 정렬

배열을 앞/뒤 두 그룹으로 나누어 각각 정렬한 뒤 병합하는 과정을 재귀적으로 반복하는 알고리즘.

- **시간 복잡도:** O(n log n)

```python
def merge_sort(a):
    def _merge_sort(a, left, right):
        if left < right:
            center = (left + right) // 2
            _merge_sort(a, left, center)
            _merge_sort(a, center + 1, right)

            # 앞부분을 버퍼에 복사 후 병합
            p = j = 0; i = k = left
            
            while i <= center:
                buff[p] = a[i]; p += 1; i += 1
            
            while i <= right and j < p:
                if buff[j] <= a[i]: a[k] = buff[j]; j += 1
                else: a[k] = a[i]; i += 1
                k += 1
            
            while j < p:
                a[k] = buff[j]; k += 1; j += 1

    buff = [None] * len(a)
    _merge_sort(a, 0, len(a) - 1)
```

# 힙 정렬

힙(Heap) 자료구조의 특성을 이용한 정렬<br/> 
힙은 **부모 노드의 값이 항상 자식 노드보다 큰** 완전 이진 트리다.

- **시간 복잡도:** O(n log n)

**정렬 과정:**
1. 배열 전체를 힙으로 만든다.
2. 루트(최댓값)를 배열 마지막 원소와 교환한다.
3. 범위를 줄인 나머지 배열을 다시 힙으로 만든다.
4. 범위가 0이 될 때까지 2~3을 반복한다.

```python
def heap_sort(a):
    def down_heap(a, left, right):
        temp, parent = a[left], left
        while parent < (right + 1) // 2:
            cl = parent * 2 + 1; cr = cl + 1
            child = cr if cr <= right and a[cr] > a[cl] else cl
            if temp >= a[child]: break
            a[parent] = a[child]; parent = child
        a[parent] = temp

    n = len(a)
    for i in range((n - 1) // 2, -1, -1):
        down_heap(a, i, n - 1)
    for i in range(n - 1, 0, -1):
        a[0], a[i] = a[i], a[0]
        down_heap(a, 0, i - 1)
```

# 도수 정렬

원소의 대소 비교 없이 각 값의 등장 횟수를 세어 정렬하는 알고리즘<br/> 
**분포수 세기 정렬**이라고도 한다.

### 3단계 과정 (예: `[5, 7, 0, 2, 4, 10, 3, 1, 3]`)

| 단계 | 설명 | 결과/공식 |
| :--- | :--- | :--- |
| **1. 도수 분포표** | 각 값이 몇 번 등장하는지 카운트하여 배열 `f`에 저장 | `f = [1,1,1,2,1,1,0,1,0,0,1]` |
| **2. 누적 도수 분포표** | 배열 `f`를 앞에서부터 누적 합산 (누적 등수 계산) | `f = [1,2,3,5,6,7,7,8,8,8,9]` |
| **3. 작업용 배열 생성** | 원본 `a`를 뒤에서부터 역순 스캔하며 목적지 배열 `b`에 배치 | `f[a[i]] -= 1`<br>`b[f[a[i]]] = a[i]` |

## 알고리즘 비교

| 알고리즘 | 평균 시간복잡도 | 특징 |
|----------|--------------|------|
| 셸 정렬 | O(n¹·²⁵) | 삽입 정렬 개선 |
| 퀵 정렬 | O(n log n) | 평균 가장 빠름, 최악 O(n²) |
| 병합 정렬 | O(n log n) | 안정 정렬, 추가 메모리 필요 |
| 힙 정렬 | O(n log n) | 추가 메모리 불필요 |
| 도수 정렬 | O(n + k) | 비교 없음, 정수형 데이터에 적합 |
