**2장 기본 자료구조와 배열**

**02-1 자료구조와 배열**  
**🔍 배열 : 묶음 단위로 값을 저장하는 것**

* 리스트(list)  
- 뮤터블 : 원소 변경 가능  
    
* 튜플(tuple)  
- 이뮤터블 : 원소 변경 불가능

\*\* 원소 변경이 불가능한 튜플도 빈 값 생성이 가능하다. 언제 활용될까?  
✔ 초기값, 불변 객체, 일관성 용도로 활용  
✔ None과 다르게 “비어있는 컬렉션” 의미  
✔ 호출 예외 처리 단순화, 사이드 이펙트 및 함수간 데이터 오염 방지  
✔ 안전한 기본값 \+ 타입 보장 \+ 불변성 확보 역할

* 리스트와 튜플의 언팩(unpack)  
  → 좌변에 변수, 우변에 리스트나 튜플을 놓으면 우변의 원소를 좌변의 변수에 한번에 대입할 수 있다.  
  → x \= \[1, 2, 3\]  
  → a, b, c \= x  
  → a, b, c  
  → (1, 2, 3\)

용어정리

| 인덱스 \[i\] | 리스트나 튜플의 원소에 접근하는 식별값. 0부터 시작 |
| :---- | :---- |
| 슬라이스 \[i:j\] | 원소의 일부를 연속하여, 혹은 일정한 간격으로 꺼내기 위한 범위식 |
| 누적 변수 | 변수에 특정값을 더한 결과를 다시 대입하여 업데이트한 값. ex) n++ |

**🔍자료구조 : 논리적인 관계로 이루어진 데이터 구성**  
→ 데이터 단위와 데이터 자체 사이의 물리적 또는 논리적인 관계  
→ 데이터가 모여있는 구조, 데이터를 효율적으로 관리하고 구조화하기 위해 파악해야 함

용어정리

| 등가성 | 두 객체의 값이 같은가? (== 비교) |
| :---- | :---- |
| 동일성 | 두 객체의 값과 식별 번호가 같은가? (is 비교) |
| 내포 표기 생성 | 리스트 안에서 for, if문을 사용하여 새로운 리스트를 생성하는 기법 |

\*\* 79p 파이썬의 모듈 재사용 보충설명  
**→**  파이썬의 모듈은 생성과 동시에 "\_\_name\_\_" 변수에 기본값으로 "\_\_main\_\_"을 갖게 된다.  
→ 직접 실행하면 : `__name__ = "__main__"`   
→ 다른 파일에서 import되면 : `__name__ = "파일명"`   
→ 즉,  `__name__` : 이 파일이 직접 실행됐는지, import됐는지 알려주는 값  
→ 함수의 재사용이라는 측면에서 js import와 작동 방식은 비슷하나, 다음과 같은 차이가 있다

| 개념 | Python | Java Script |
| :---: | ----- | ----- |
| 실행 구분 | \_\_name\_\_로 자동 | 별도 코드 필요 |
| import시 실행 | 가능하지만 제어 가능 | 무조건 실행 |
| 모듈 목적 | 실행 \+ 라이브러리 겸용 | 모듈 중심 |

**🔍 파이썬의 매개변수**  
→ 파이썬에서는 매개변수에 실제 인수가 대입된다. 즉, 실제 값 그 자체가 할당되는것이 아닌 객체 참조에 의한 전달이 이루어지는 것. 쉽게 말해, 객체를 넘기는 것이 아닌 객체를 가리키는 참조를 복사해서 넘긴다.
