# 검색 알고리즘이란?

데이터 집합에서 특정 조건(키)을 만족하는 원소를 찾아내는 과정이다.

* **키(key):** 검색의 조건이 되는 항목 (예: 국적이 한국인 사람, 이름에 '민'자가 들어간 사람 등)
* **데이터:** 키와 연결된 실제 정보 값

<br/><br/>

## 검색의 종류

### 1) 배열 검색
* **1-1) 선형 검색:** 무작위로 늘어놓은 데이터 집합에서 순차적으로 스캔하며 검색
* **1-2) 이진 검색:** 일정한 규칙으로 정렬된 데이터 집합에서 범위를 절반씩 좁히며 아주 빠르게 검색
* **1-3) 해시법:** 데이터 추가/삭제가 자주 일어나는 집합에서 아주 빠른 검색 수행
    * **체인법:** 같은 해시값 데이터를 연결 리스트로 연결하는 방법
    * **오픈 주소법:** 해시 충돌 시 재해시하여 빈 자리를 찾는 방법
### 2) 연결 리스트 검색
### 3) 이진 검색 트리 검색

> **💡 알고리즘 선택 TIP**
> 단순히 검색 속도만 중요하다면 계산 시간이 짧은 것을 선택하면 되지만, 데이터의 추가나 삭제 작업이 빈번하다면 해당 작업의 비용까지 종합적으로 평가하여 선택해야 한다.

<br/><br/>

## 선형 검색 (Linear Search)

원하는 키값을 가진 원소를 찾을 때까지 맨 앞부터 순서대로 스캔하는 알고리즘.

<br/>

### 선형 검색의 종료 조건
1.  **검색 실패:** 값을 찾지 못하고 배열의 맨 끝을 지나간 경우
2.  **검색 성공:** 검색할 값과 같은 원소를 찾은 경우

<br/>

### 보초법 (Sentinel Method)
선형 검색은 반복할 때마다 위 2가지 종료 조건을 매번 체크하는데, 이 비용을 반으로 줄이는 방법

* **원리:** 원래 데이터 끝에 찾으려는 값(**보초**)을 추가하여 저장
* **특징:** 보초 덕분에 무조건 값을 찾게 되므로 "배열의 끝을 지나갔는가?"를 판단할 필요가 없어진다.
* **마지막 처리:** `return -1 if i == len(seq) else i`
    * 찾은 위치 `i`가 원래 데이터의 길이와 같다면 보초를 찾은 것이므로 검색 실패(`-1`)를 반환

<br/><br/>

## 이진 검색 (Binary Search)

* **필수 조건:** 배열의 데이터가 반드시 **정렬**되어 있어야 함
* **시간 복잡도:** $O(\log n)$

<br/><br/>

## 복잡도 (Complexity)

1.  **시간 복잡도:** 알고리즘을 실행하는 데 필요한 **시간(연산 횟수)** 을 평가하는 것
2.  **공간 복잡도:** 메모리(기억 공간)와 파일 공간이 얼마나 필요한지 평가하는 것
