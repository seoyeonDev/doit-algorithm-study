# 검색 알고리즘
- 키 : 주목하는 항목, 대부분 데이터의 일부

## 배열 검색
### 선형 검색
- 직선 모양(선형) 으로 늘어선 배열에서 검색하는 경우에 **원하는 키 값을 가진 원소를 찾을 떄까지 맨 앞부터 스캔** 하여 순서대로 검색
- 원소 값이 정렬되지 않은 배열에서 검색할때 사용하는 유일한 방법
- 원소 개수 n 개, 조건 판단 횟수 평균 n/2번 
```
찾는 값이 운 좋으면 가장 처음(1), 운 나쁘면 가장 마지막(n), 정중앙 (n/2)
수학적으로 계산 시 
(1 + 2 + 3 + ... + n) / n = n(n+1)/2 / n = (n+1)/2
즉, 어디 있을 지 모르는 걸 처음부터 흝으면 평균적으로 절반쯤 가서 만난다.

```
- 시간 복잡도 O(n)

#### 보초법
- 검색하고자 하는 키값을 배열의 맨 끝에 저장.
- 보초 : 저장하는 값
<br/><br/>
검색할 값과 같은 원소를 찾았습니까? 만 종료 조건으로 성립하여, **종료 조건을 검사하는 비용 절감**


### 이진 검색
- 배열의 데이터가 정렬되어 있어야 사용 가능
- pl : 가장 첫 값 , pc: (n-1)/2 , pr : n - 1
- a[pc] < key : pc 뒤쪽 값으로 범위 좁혀짐 
- a[pc] > key : pc 앞쪽 값으로 범위 좁혀짐

- 반복할 때마다 검색 범위가 거의 절반으로 줄어듦.
- 소요되는 비교 횟수는 `log n`  
- 시간 복잡도 O(log n)

#### 복잡도
1. 시간 복잡도
```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
```
- 전체 복잡도는 차원이 가장 높은 복잡도 선택하는 것
