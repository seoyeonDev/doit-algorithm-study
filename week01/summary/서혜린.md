**1장 알고리즘 기초**

**01-1 알고리즘이란?**  
→ 어떠한 문제를 해결하기 위해 정해 놓은 일련의 절차  
→ 올바른 알고리즘은 어떠한 경우에도 실행 결과가 똑같이 나오는 것

용어정리

| 순차 구조 | 한 문장씩 순서대로 처리되는 구조 |
| :---- | :---- |
| 선택 구조 | 조건식으로 평가한 결과에 따라 프로그램의 실행 흐름이 변경되는 구조 |
| 양 갈래 선택 | 작성한 조건식에 따라 알고리즘 흐름이 두 갈래로 나뉘는 것 |
| 결정 트리 | 조합의 나열, 경우의 수 |

* 코드가 짧으면 효율적일 것이라는 착각에 빠지지 말 것. 중요한 건 이미 검증된 조건을 반복 검증하지 않도록 알고리즘을 구성하는 것 (25p 참고)

**01-2 반복하는 알고리즘**

* **while문**  
  → 어떤 조건이 성립하는 동안 반복 (조건이 false 되면 종료)

* **for문**   
  → 변수가 하나만 있는 경우의 반복 (리스트, 문자열, 범위(range) 같은 반복 횟수가 명확한 객체를 순회)

용어정리

| 이터러블 객체 | 반복할 수 있는 객체. list, str, tuple 등이 있음. |
| :---- | :---- |
| 무한 루프 | 의도적으로 while문이 무한 반복되도록 만든 것 |
| 구조적 프로그래밍 | 순차·조건·반복 같은 기본 제어 구조만으로 프로그램을 구성하는 방식  |
| 다중 루프 | 반복문 안에 반복문을 중첩 사용하는 구조 |

**\*\* 58p 파이썬의 객체 할당 방식 보충설명**  
→ C, java와 달리 파이썬은 정수를 프로세스 시작 시 미리 생성해서 캐싱함. 즉, `id()`가 다른 건 "다른 객체를 가리킨다"는 의미지, 매번 새로 할당(생성)된다는 의미가 아님. 그래서 i 에 할당된 1\~100의 값에 대한 식별번호를 “꺼내온다”고 표현한 것이기는 하나 좀 헷갈릴 수 있을듯 하여 보충함.
