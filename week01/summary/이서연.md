# 요약
- 부동 소수점 : 
    - float 가 사용, 
    - 소수점의 위치가 고정되지 않고 숫자의 크기에 따라 움직임. 
    - 유효한 숫자 부분 + 소수점의 위치 부분

- 알고리즘 :
    - 어떠한 문제를 해결하기 위해 정해 놓은 일련의 절차 
    - 어떠한 경우에도 실행 결과가 똑같이 나오는 것 

- 파이썬 스타일 가이드 PEP 8 : 
    - 클래스명 카멜 케이스, 함수명 스네이크, 들여쓰기 시 공백 4개 사용 권장 

- if 문 실행에 대한 효율화. (38p)
    - 굳이 필요하지 않게 여러번 반복되는 경우 제외하여 별도로 두는 것이 좋음.

- 구조적 프로그래밍이 아닌 것.
    - 선언형 프로그래밍, 무엇을 할지만 선언 
    - SQL(DB가 알아서 찾음), HTML(구조만 선언), CSS
    - 반복문도, 조건문도 내가 안 써도 되는 것. 

- 파이썬의 변수
    - 파이썬의 변수는 값을 갖지 않음. 객체 취급
    - _모든 객체가 메모리를 가지고 있으면 메모리 점유율이 너무 높은거 아닌지?_ (58p 파이썬에서는 함수가 시작하고 종료함에 따라 객체가 생성되거나 소멸하지 않음)
        - 함수가 끝나면 기본적으로 객체 소멸
        - 각 객체가 몇 번 참조되고 있는지 숫자로 셈. (순환 참조의 경우 가비지 컬렉터가 주기적으로 청소)
        - 작은 정수, 짧은 문자열은 파이썬이 미리 만들어두고 재사용 
    - 프로그램에서 1이라는 정수 객체는 put_id() 함수와 무관하게 존재.
        - 인터닝 때문 : 파이썬은 자주 쓰는 정수를 미리 만들어 둠.

# 개념
## 순차 구조
- 한 문장씩 순서대로 처리되는 구조

## 조건식
- if 와 콜론 사이에 있는 식 
- 조건식은 마름모 모양으로 표현
-> 조건식에 따라 알고리즘 흐름이 두 갈래로 나뉘는 것 : **양갈래 선택**

## 선택 구조
- 조건식으로 평가한 결과에 따라 프로그램의 실행 흐름이 변경 

## 복합문의 구조
- 헤더 : if, while 키워드로 시작, 콜론(:) 으로 끝남 
- :은 바로 뒤에 스위트가 이어집니다 를 의미 
- 반드시 행마다 같은 수준으로 들여쓰기 
- 스위트가 단순문이면 헤더와 같은 행 가능
- 단순문이 2개 이상이면 각각의 단순문을 세미콜론(;) 으로 구분하여 헤더와 같은 행 가능
- *스위트가 복합문이면 헤더와 같은 행 X*

```
if : 스위트 
elif : 스위트 
else : 스위트 
```

## 결정 트리
- 왼쪽 (a >= b) ~> 오른쪽으로 나아감 


## 연산자
### 산술 연산자 
- +, - 등의 기호

### 피연산자
- 연산 대상 

### 연산자 종류
- 단항 연산자 : 피연산자 1개
- 이항 연산자 : 피연산자 2개
- 삼항 연산자 : 피연산자 3개 

### 조건 연산자
- if ~ else 문은 파이썬의 유일한 삼항 연산자 


## 순서도 
- 문제를 정의, 분석하고 해결하는 방법을 그림으로 표현 

## 이터러블 객체
- 반복할 수 있는 객체


## 드모르간 법칙
- 논리 연산에서 not 을 분배할 때 and <-> or 이 뒤바뀐다는 법칙
- not (A and B) = (not A) or (not B)
- not (A or B) = (not A) and (not B)
